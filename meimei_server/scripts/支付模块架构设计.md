# 支付模块架构设计文档

## 📋 模块概述

支付模块是一个独立的业务模块，采用**策略模式**实现多支付通道支持，具备**重试机制**和**通道切换**能力。

## 🎯 核心设计模式

### 1. 策略模式（Strategy Pattern）

**目的**：支持多种支付通道，易于扩展

**核心接口**：
```typescript
interface PaymentStrategy {
  createPayment(params): Promise<CreatePaymentResult>;
  queryPayment(paymentNo): Promise<QueryPaymentResult>;
  verifyNotify(data): boolean;
  handleNotify(data): Promise<NotifyResult>;
  supportCurrency(currency): boolean;
  supportMethod(method): boolean;
  getPriority(): number;
}
```

**抽象基类**：
```typescript
abstract class AbstractPaymentStrategy implements PaymentStrategy {
  protected channelConfig: ChannelConfig;
  protected httpService: HttpService;
  protected logger: Logger;
  
  // 实现通用逻辑
  // 子类实现特定逻辑
}
```

### 2. 工厂模式（Factory Pattern）

**目的**：根据通道编码创建对应的策略实例

```typescript
@Injectable()
export class PaymentStrategyFactory {
  private strategies: Map<string, PaymentStrategy> = new Map();
  
  registerStrategy(strategy: PaymentStrategy) {
    this.strategies.set(strategy.getChannelCode(), strategy);
  }
  
  getStrategy(channelCode: string): PaymentStrategy {
    return this.strategies.get(channelCode);
  }
}
```

### 3. 重试机制（Retry Pattern）

**目的**：支付失败时自动重试

```typescript
async executeWithRetry(
  paymentNo: string,
  maxRetry: number,
  operation: () => Promise<any>
): Promise<any> {
  let lastError: Error;
  
  for (let i = 0; i <= maxRetry; i++) {
    try {
      return await operation();
    } catch (error) {
      lastError = error;
      await this.updateRetryCount(paymentNo, i);
      
      if (i < maxRetry) {
        await this.delay(5000); // 等待5秒
      }
    }
  }
  
  throw lastError;
}
```

### 4. 通道选择策略

**目的**：按优先级和信用卡支持自动选择通道

```typescript
async selectPaymentChannel(
  amount: number,
  currency: string,
  paymentMethod: string
): Promise<PaymentChannel> {
  // 1. 查询所有启用的通道
  const channels = await this.channelService.findActiveChannels();
  
  // 2. 过滤支持的通道
  const supported = channels.filter(ch => 
    ch.supportedCurrencies.includes(currency) &&
    ch.supportedMethods.includes(paymentMethod) &&
    amount >= ch.minAmount &&
    amount <= ch.maxAmount
  );
  
  // 3. 按优先级排序（数字越小优先级越高）
  supported.sort((a, b) => a.priority - b.priority);
  
  // 4. 优先选择信用卡通道
  const creditCardChannel = supported.find(
    ch => ch.channelType === 'CREDIT_CARD'
  );
  
  return creditCardChannel || supported[0];
}
```

## 🔄 业务流程

### 1. 创建支付订单流程

```
用户请求 /checkout/pay?v=ORDER123
         ↓
PaymentController.getPayPage(orderNo)
         ↓
PaymentService.createPayment(orderNo)
         ↓
1. 查询订单信息
2. 查询配送信息（biz_deliver表）
3. 选择支付通道（策略模式）
4. 创建支付订单记录
5. 调用第三方支付API
6. 返回支付链接/二维码
         ↓
返回支付页面
```

### 2. 支付回调流程

```
第三方支付平台 POST /api/payment/notify/xpay
         ↓
NotifyController.handleNotify(channelCode, data)
         ↓
1. 验证签名
2. 记录回调日志
3. 更新支付订单状态
4. 更新业务订单状态
5. 返回成功响应
```

### 3. 重试机制流程

```
支付失败
   ↓
检查重试次数 < max_retry
   ↓ 是
等待 retry_interval 秒
   ↓
重新发起支付
   ↓
记录重试日志
   ↓ 否
标记支付失败
通知用户
```

## 📊 数据库设计

### 核心表关系

```
biz_order (业务订单)
   ↓ 1:1
biz_deliver (配送信息)
   ↓ 1:N
biz_payment_order (支付订单)
   ↓ N:1
biz_payment_channel (支付通道)
   
biz_payment_notify_log (回调日志)
   ↓ N:1
biz_payment_order (支付订单)
```

### 关键字段说明

**biz_payment_order**:
- `payment_no`: 系统生成的支付订单号（唯一）
- `order_no`: 关联业务订单号
- `status`: 支付状态（0-5）
- `retry_count`: 已重试次数
- `max_retry`: 最大重试次数

**biz_payment_channel**:
- `channel_code`: 通道唯一编码
- `channel_type`: 通道类型（CREDIT_CARD/QRIS/ALIPAY）
- `priority`: 优先级（数字越小越优先）
- `is_active`: 是否启用

## 🔐 X支付通道实现

### 1. 签名算法

```typescript
generateSign(params: Record<string, any>, secret: string): string {
  // 1. 参数按字母排序
  const sortedKeys = Object.keys(params).sort();
  
  // 2. 拼接字符串
  const signStr = sortedKeys
    .map(key => `${key}=${params[key]}`)
    .join('&') + `&key=${secret}`;
  
  // 3. MD5加密并转大写
  return crypto
    .createHash('md5')
    .update(signStr)
    .digest('hex')
    .toUpperCase();
}
```

### 2. 创建支付订单

```typescript
async createPayment(params: CreatePaymentParams): Promise<CreatePaymentResult> {
  const requestData = {
    platform_key: this.channelConfig.platformKey,
    site_code: this.channelConfig.siteCode,
    order_no: params.orderNo,
    amount: Math.round(params.amount * 100), // 转为分
    currency: params.currency,
    payment_method: params.paymentMethod,
    timestamp: Math.floor(Date.now() / 1000),
    notify_url: params.notifyUrl,
  };
  
  // 生成签名
  requestData.sign = this.generateSign(requestData, this.channelConfig.platformSecret);
  
  // 调用API
  const response = await this.httpService.post(
    `${this.channelConfig.apiBaseUrl}/api/${this.channelConfig.apiVersion}/pay/create`,
    requestData
  ).toPromise();
  
  // 处理响应
  if (response.data.code === 0) {
    return {
      success: true,
      paymentNo: params.orderNo,
      thirdPaymentNo: response.data.data.third_order_no,
      payUrl: response.data.data.pay_url,
      qrCode: response.data.data.qr_code,
      expireTime: new Date(response.data.data.expire_time),
      rawData: response.data,
    };
  }
  
  return {
    success: false,
    paymentNo: params.orderNo,
    errorMsg: response.data.msg,
  };
}
```

## 🎓 扩展新支付通道

1. 在 `biz_payment_channel` 表添加通道配置
2. 创建新的策略类，继承 `AbstractPaymentStrategy`
3. 实现必要的方法
4. 在 `PaymentStrategyService` 中注册策略

**示例**：
```typescript
@Injectable()
export class NewPayStrategy extends AbstractPaymentStrategy {
  getChannelCode(): string {
    return 'NEW_PAY';
  }
  
  // 实现其他方法...
}
```

## ⚠️ 注意事项

1. **签名安全**：所有签名必须验证，防止伪造
2. **幂等性**：回调处理必须支持幂等，防止重复处理
3. **重试机制**：重试次数和间隔可配置
4. **日志记录**：所有支付操作必须记录日志
5. **事务管理**：支付状态更新必须使用事务
6. **异常处理**：捕获所有异常，记录详细错误信息
